# Office — 자율 에이전트 오피스 게임 UI

38명의 AI 에이전트가 직원으로 부서별 책상에 앉아 능동적으로 업무를 수행하는 아이소메트릭 오피스 뷰. 좌측 사이드바에는 KPI·승인 대기·진행 업무·활동 피드, 우측 메인 씬에는 캐릭터들이 자리 잡고 말풍선·상태 애니메이션으로 실시간 활동을 표시.

## 실행

이 UI는 단독 정적 사이트가 아니라 `server/`의 백엔드와 함께 동작.

```bash
cd /home/user/AI_maketing_ofice
node server/server.mjs       # 한 명령으로 백엔드 + 정적 서빙
# → http://localhost:8787
```

실제 Claude API 연동은 `ANTHROPIC_API_KEY` 환경변수만 설정하면 활성화됨. 키가 없으면 시뮬레이션 모드로 동일한 UI 흐름 확인 가능. 자세한 백엔드 설정은 [`../server/README.md`](../server/README.md).

## 화면 구성

### 좌측 사이드바
- **KPI 4종**: 직원 / 업무중 / 대기 / 승인대기 (실시간)
- **🛎 승인 대기**: 금전·업로드 등 중요 이벤트로 인해 멈춘 작업. 카드 클릭 → 모달에서 **에이전트 작업 결과** 검토 후 승인/거절
- **⚙️ 진행 중 업무**: queued/working/awaiting 상태 모든 태스크 실시간 표시
- **📜 활동 피드**: 최근 60개 이벤트 (시작·완료·승인·거절·실패)
- **+ 수동 업무 생성**: 무작위 에이전트에게 즉시 태스크 부여 (테스트용)

### 메인 씬 (오피스)
- 8개 부서 구역 (전환율 최적화·콘텐츠 카피·SEO 발견·유료 광고·그로스 리텐션·세일즈 GTM·전략 리서치·한국 특화)
- 각 부서마다 색상 라벨, 소속 에이전트들의 책상
- 책상마다 캐릭터(부서 색·이모지 얼굴·KO 배지) + 책상 + 모니터
- 캐릭터 상태:
  - 🟢 **idle**: 차분히 대기
  - 🔵 **working**: 몸이 흔들리며 ⚙️ 회전 인디케이터
  - 🟡 **awaiting**: 노란 펄스 후광 + ⚠️ 흔들림
- 말풍선: 작업 시작 시 업무 제목, 완료 시 ✅ 완료, 승인 필요 시 ⚠️ 승인 필요, 실패 시 ⚠️ 실행 실패

## 파일

```
office/
├── index.html        # 사이드바 + 씬 + 모달 레이아웃
├── style.css         # 2D 스큐 아이소메트릭 + 캐릭터 스타일·애니메이션
├── characters.js     # 부서별 책상 그리드 빌드, 캐릭터 DOM 렌더, 말풍선 관리
└── engine.js         # SSE 클라이언트, 상태 동기화, 사이드바 렌더, 승인 모달, 수동 트리거
```

스킬 메타데이터 자체는 `dashboard/data/skills.json` 공유 (서버가 `/data/skills.json`로 노출).

## 자동화 흐름

서버가 시작되면 즉시:

1. 시드 백로그 6개 태스크가 무작위 에이전트들에게 분배됨 → 캐릭터들이 working 상태로 전환, 말풍선에 업무 제목 표시
2. 15초마다 idle 에이전트 한 명에게 새 태스크 자동 부여
3. 태스크가 정책상 승인 필요(`paid-ads`, `email-sequence`, `naver-kin-automation` 답변 등록 등)면 `awaiting_approval` 상태로 멈추고 사이드바에 빨간 카드 등장
4. 사용자가 카드 클릭 → 모달에서 "이유", "제안된 행동", "에이전트 작업 결과" 검토 → 승인/거절
5. 승인 시 즉시 `completed`, 거절 시 `rejected`. 활동 피드에 기록.

## 승인 게이트 정책 (요약)

- 💰 금전 이벤트: 광고비 집행, 가격 변경, 페이월 변경, 레퍼럴 보상률 변경
- 📤 대외 업로드: 이메일 발송, 소셜 게시, 광고 크리에이티브 배포, 영상 발행, 지식iN 답변 등록, 런칭 발표
- 🏗️ 구조 변경: 사이트 IA 변경, 스키마 배포, 대량 페이지 생성·발행
- 🚪 고객 경험 영향: 해지 플로우 변경

위 외 모든 스킬(SEO 감사, 카피 초안, 고객 리서치, 분석 리포트 등)은 **승인 없이 자율 실행**.

세부 정책은 [`../server/policies.mjs`](../server/policies.mjs).

## 실시간 통신

- 전송: 클라이언트가 `/api/stream` (Server-Sent Events) 구독
- 이벤트: `task.created`, `task.updated`, `approval.requested`, `approval.resolved`, `agent.status`
- 첫 연결 시 `event: snapshot`로 전체 상태 한 번 동기화
- 25초마다 keepalive ping
- 연결 끊기면 브라우저가 자동 재연결

## 한계 / Phase 3 후보

- 캐릭터 위치는 부서 그리드 고정 — 자유 이동·휴게실 방문 같은 동선 표현은 없음
- 픽셀 아트가 아닌 SVG/이모지 — 진짜 게임 비주얼은 별도 아트 자산 필요
- 에이전트 간 상호작용 없음 — 미래 후보: 한 스킬 결과를 다른 스킬 input으로 전달하는 멀티 에이전트 워크플로우
- 태스크 큐는 in-memory — 운영 시 영속성 필요
- 승인 거절 시 재시도 플로우 없음 — 거절되면 rejected로 종료
- 모바일 뷰는 좁음(960px↓에서 사이드바 상단 50vh)
