# 모니터링 검색어 템플릿

지식iN에서 자사 관련 질문과 브랜드 언급을 체계적으로 수집하기 위한 검색어·API 쿼리 설계.

## 1. 검색 계층

3계층으로 검색어를 설계한다.

| 계층 | 목적 | 빈도 |
|------|------|------|
| L1: 브랜드 직접 | 자사·경쟁사 언급 탐지 | 매일 |
| L2: 카테고리 키워드 | 답변 기회 발굴 | 매일 |
| L3: 롱테일·상황 키워드 | 고가치 기회 발굴 | 주간 |

## 2. L1: 브랜드 모니터링

### 2-1. 자사 브랜드

```
"<공식 브랜드명>"                     # 정확 매칭
<브랜드명 약칭>                       # 예: "토스" (토스페이먼츠 축약)
<브랜드명 오탈자 변형>                # 예: "챗지피티", "Chat GPT"
<제품명 + 핵심 기능>                  # 예: "노션 데이터베이스"
```

### 2-2. 경쟁사 브랜드

자사 context 기반 경쟁사 리스트 작성. 각 경쟁사마다 L1과 동일 패턴 적용.

### 2-3. 결과 분류

수집된 언급을 즉시 다음으로 분류.

- **긍정**: 질문자가 브랜드를 호의적으로 언급 → 내부 공유, 마케팅 자산화 고려
- **중립**: 단순 비교 질문 → 답변 기회 (간접 전략 적용)
- **부정**: 불만·문제 제기 → 고객 성공팀 즉시 이관 (지식iN 내 대응 X, 공식 채널로 연결)
- **비방·허위**: 법무팀 이관 (삭제 요청 검토)

## 3. L2: 카테고리 키워드

업계 일반 질문 수집.

### 3-1. B2B SaaS 예시

```
"CRM 추천"
"이메일 마케팅 툴"
"고객 데이터 분석"
"마케팅 자동화"
"세일즈 파이프라인"
"구독 결제 시스템"
```

### 3-2. 이커머스 예시

```
"스마트스토어 운영"
"온라인 쇼핑몰 창업"
"해외 배송 업체"
"상품 사진 촬영"
"고객 응대 매뉴얼"
```

### 3-3. 부동산 예시

```
"전세 계약 주의사항"
"임대차 보호법"
"공인중개사 수수료"
"전월세 전환율"
```

### 3-4. 템플릿

```
"<제품 카테고리> 추천"
"<업계> 시작 방법"
"<기능> 어떻게"
"<문제 상황> 해결"
```

## 4. L3: 롱테일·상황 키워드

구체적 상황 + 감정 표현 + 의문형. 답변이 채택될 확률이 가장 높다.

```
"초보자가 <action> 하려는데"
"<상황>인데 <문제>가 생겼어요"
"<제품>을 써봤는데 <불만>이에요"
"<예산>으로 <목표> 가능한가요?"
"<A> vs <B> 뭐가 나은가요"
```

실제 예시:
- "초보자가 쇼핑몰 창업하려는데 뭐부터 해야 하나요"
- "월 100만원 예산으로 퍼포먼스 광고 어디가 나을까요"
- "노션이랑 에어테이블 중에 어떤 게 B2B 세일즈용으로 나을까요"

## 5. 네이버 검색 오픈API 호출

지식iN 공식 API는 없고, 검색 오픈API로 지식iN 결과를 받는다.

### 5-1. 엔드포인트

```
GET https://openapi.naver.com/v1/search/kin.json
  ?query=<검색어>
  &display=10        # 1-100
  &start=1           # 1-1000
  &sort=sim          # sim(정확도) | date(날짜)
```

인증: 헤더에 `X-Naver-Client-Id`, `X-Naver-Client-Secret`.

### 5-2. 응답 스키마 (요약)

```json
{
  "items": [
    {
      "title": "질문 제목 (HTML escape 필요)",
      "link": "https://kin.naver.com/qna/detail.nhn?...",
      "description": "질문 본문 요약",
      "lastBuildDate": "Mon, 14 Apr 2026 10:00:00 +0900"
    }
  ]
}
```

### 5-3. 레이트 리밋

네이버 오픈API는 앱당 일 25,000건 기본 제공. 지식iN 모니터링 규모에는 충분하지만, L1×3 + L2×15 + L3×30 × 일 2회 polling = 96회/일 수준으로 여유 있게 설계.

### 5-4. 모니터링 스크립트 구조 (의사 코드)

```js
// tools/clis/naver-kin.js 참고 구현 힌트 (스킬 본체에는 자동 실행 없음)
const queries = {
  L1: ['자사브랜드', '경쟁사A', '경쟁사B'],
  L2: ['CRM 추천', '이메일 마케팅 툴', /* ... */],
  L3: ['초보자가 CRM', 'B2B SaaS vs', /* ... */]
};
for (const [tier, qs] of Object.entries(queries)) {
  for (const q of qs) {
    const res = await fetch(openapiUrl + '?query=' + encodeURIComponent(q), { headers });
    const { items } = await res.json();
    writeToCSV(tier, q, items);   // 사람이 검토할 CSV/구글시트에 축적
  }
}
```

CLI 상세는 `tools/clis/naver-kin.js` (Phase 5 T5에서 선택 구현) 참조.

## 6. 결과 통합 관리

### 6-1. 리포팅 포맷 (구글시트·Notion)

| 열 | 예시 |
|-----|------|
| 수집일 | 2026-04-18 |
| Tier | L1 / L2 / L3 |
| 검색어 | "CRM 추천" |
| 질문 URL | https://kin.naver.com/qna/... |
| 질문 제목 | |
| 질문 요약 | |
| 기존 답변 수 | 2 |
| 채택 여부 | No |
| 스코어 | 0-100 |
| 대응 후보? | Yes / No |
| 담당자 | |
| 답변 상태 | 초안 / 검토 / 등록 / 채택 / 폐기 |
| 감성 | 긍정·중립·부정·비방 |

### 6-2. 주간 리뷰 체크리스트

- [ ] L1 탐지된 자사 언급 모두 분류·대응했는가?
- [ ] L1 탐지된 경쟁사 언급 패턴 인사이트가 있는가?
- [ ] L2/L3 스코어 60+ 질문을 몇 건 답변했는가?
- [ ] 답변 채택률 (채택 ÷ 등록)이 30% 이상인가?
- [ ] 정책 위반 신고 없이 한 주를 보냈는가?
